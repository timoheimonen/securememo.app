À propos de securememo.app

securememo.app est une application axée sur la confidentialité qui vous permet de créer des mémos chiffrés qui s'autodétruisent après avoir été lus ou expirés. Construite avec la sécurité et la confidentialité comme priorités principales. Code source disponible sur GitHub.

Stack Technologique

  • Go service: Self-hosted HTTP service
  • SQLite database: SQLite-powered database for secure storage
  • Web Crypto API: Web Crypto API : Chiffrement AES-256 côté client
  • HTML/CSS/JavaScript: HTML/CSS/JavaScript : Interface web moderne et responsive
  • Code source disponible sur GitHub : Voir le code source sur GitHub

Fonctionnalités de Sécurité

🔐 Chiffrement Côté Client

Tout le chiffrement se produit dans votre navigateur en utilisant l'API Web Crypto. Votre mémo est chiffré avec AES-256 avant d'être envoyé à nos serveurs.

🔑 Partage de Mot de Passe Séparé

Le mot de passe de chiffrement est généré séparément de l'URL et devrait être partagé via un canal différent pour une sécurité renforcée.

💥 Mémos Auto-Destructeurs

Une fois qu'un mémo est lu ou expiré, il est immédiatement et définitivement supprimé de notre base de données. Aucune trace ne demeure.

🌐 Aucun Stockage de Mot de Passe

Nous ne stockons jamais les mots de passe de chiffrement sur nos serveurs. Ils n'existent que dans la mémoire de votre navigateur.

Self-hosted service

Runs as a Go service backed by SQLite, suitable for deployment behind a reverse proxy or tunnel.

🔒 Confidentialité d'Abord

Aucun compte utilisateur requis. Aucun suivi. Aucune analytique. Votre confidentialité est notre priorité.

Comment Utiliser

1. Créer un Mémo

Allez à "Créer un Mémo Sécurisé" et tapez votre mémo. Choisissez un temps d'expiration (8h, 1j, 2j, 1 semaine ou maximum 30 jours). Les mémos sont supprimés quand ils sont lus ou expirés. Le système générera un mot de passe de chiffrement aléatoire.

2. Partager le Lien et le Mot de Passe

Copiez l'URL et le mot de passe générés, puis partagez-les avec votre destinataire via des canaux séparés pour une sécurité renforcée.

3. Auto-Destruction

Quand le destinataire ouvre le lien et saisit le mot de passe séparément, le mémo sera déchiffré puis supprimé définitivement s'il est lu, ou supprimé automatiquement s'il expire. Les mémos expirés sont automatiquement nettoyés toutes les heures.
