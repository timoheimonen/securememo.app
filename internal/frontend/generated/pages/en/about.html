About securememo.app

securememo.app is a privacy-focused application for creating encrypted memos that are deleted after successful read confirmation or expiration. Built with security and privacy as priorities. Source code available on GitHub.

Security Features

🔐 Client-Side Encryption

All encryption happens in your browser using the Web Crypto API. Your memo is encrypted with AES-256 before being sent to our servers.

🔑 Separate Password Sharing

The encryption password is generated separately from the URL and should be shared through a different channel for enhanced security.

Automatic Deletion

After a memo is successfully read, the browser confirms deletion with the server. Expired memos are removed automatically during cleanup.

🌐 No Password Storage

We never send or store encryption passwords on our servers. The password is generated and handled in the browser.

Service backend

The service uses a Go backend with SQLite for temporary encrypted memo storage.

🔒 Privacy First

No user accounts, no analytics, and no marketing tracking. Limited operational data is used only to run and protect the service.

How to Use

1. Create a Memo

Go to "Create Secure Memo" and type your memo. Choose an expiry time (8h, 1d, 2d, 1 week or a maximum of 30 days). Memos are deleted after successful read confirmation or expiration. The browser will generate a random encryption password.

2. Share the Link and Password

Copy the generated URL and password, then share them with your recipient through separate channels for enhanced security.

3. Automatic deletion

When the recipient opens the link and enters the password separately, the memo is decrypted in the browser. After a successful read, the browser asks the server to delete it; expired memos are cleaned up automatically every hour.
