securememo.app 소개

securememo.app은 읽거나 만료된 후 자체 삭제되는 암호화된 메모를 생성할 수 있는 개인정보 보호 중심 애플리케이션입니다. 보안과 개인정보 보호를 최우선으로 구축되었습니다. 소스 코드는 GitHub에서 확인할 수 있습니다.

기술 스택

  • Go service: Self-hosted HTTP service
  • SQLite database: SQLite-powered database for secure storage
  • Web Crypto API: Web Crypto API: 클라이언트 측 AES-256 암호화
  • HTML/CSS/JavaScript: HTML/CSS/JavaScript: 현대적이고 반응형 웹 인터페이스
  • GitHub에서 소스 코드 확인 가능: GitHub에서 소스 코드 보기

보안 기능

🔐 클라이언트 측 암호화

Web Crypto API를 사용하여 브라우저에서 모든 암호화가 발생합니다. 메모는 서버로 전송되기 전에 AES-256으로 암호화됩니다.

🔑 별도 비밀번호 공유

암호화 비밀번호는 URL과 별도로 생성되며 보안 강화를 위해 다른 채널을 통해 공유되어야 합니다.

💥 자체 삭제 메모

메모가 읽히거나 만료되면 데이터베이스에서 즉시 영구적으로 삭제됩니다. 흔적이 남지 않습니다.

🌐 비밀번호를 저장하지 않음

서버에 암호화 비밀번호를 저장하지 않습니다. 브라우저 메모리에만 존재합니다.

Self-hosted service

Runs as a Go service backed by SQLite, suitable for deployment behind a reverse proxy or tunnel.

🔒 개인정보 보호 우선

사용자 계정이 필요하지 않습니다. 추적 없음. 분석 없음. 개인정보 보호가 우선순위입니다.

사용 방법

1. 메모 작성

"보안 메모 작성"으로 이동하여 메모를 입력하세요. 만료 시간을 선택하세요(8시간, 1일, 2일, 1주일 또는 최대 30일). 메모는 읽거나 만료되면 삭제됩니다. 시스템이 무작위 암호화 비밀번호를 생성합니다.

2. 링크와 비밀번호 공유

생성된 URL과 비밀번호를 복사한 후 보안 강화를 위해 별도의 채널을 통해 수신자와 공유하세요.

3. 자체 삭제

수신자가 링크를 열고 별도로 비밀번호를 입력하면 메모가 복호화되고 읽은 경우 영구적으로 삭제되거나 만료된 경우 자동으로 삭제됩니다. 만료된 메모는 매 1시간마다 자동으로 정리됩니다.
